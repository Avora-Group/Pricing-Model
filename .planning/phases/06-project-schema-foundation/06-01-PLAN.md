<objective>
Add project lifecycle status and provenance to `pricing_projects`: a `status` column ('potential'|'signed', default 'potential'), `status_source` ('automatic'|'manual', default 'manual'), and a nullable `signed_at` timestamp. Expose all three on the `ProjectResponse` API model so downstream phases (7, 9, 10) can read them. This is pure foundation plumbing — NO status-mutation or auto-sign behavior ships here (that is Phase 7/9).

Purpose: Every v2.0 pipeline feature needs a place to record whether a project is potential or signed and whether that decision was made by a human or by automation. The provenance columns must exist NOW with the correct defaults so Phase 7's auto-sign guard (`WHERE status_source <> 'manual'`) is expressible.
Output: Migration 008, an extended `ProjectResponse`, updated route constructors, mock-DB defaults, and green tests asserting the new defaults round-trip.
</objective>
